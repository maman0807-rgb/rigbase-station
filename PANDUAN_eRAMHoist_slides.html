4. Reliability — Keandalan

Mengukur seberapa sering equipment mengalami kerusakan.

MTBF (Mean Time Between Failures)

MTBF = jam_jalan ÷ jumlah_kerusakan

Kategori dihitung sebagai event: breakdown troubleshoot

Bukan event (durasi saja): tunggu_spare

MTBF (jam)Status
< ½ siklus TOH🔴 Keandalan rendah
½ – 1 siklus TOH🟡 Pantau
≥ 1 siklus TOH🟢 Sehat

5. Availability — Kesiapan Equipment

Availability % = (Jam periode − Jam down) ÷ Jam periode × 100

KPI Dashboard (kartu atas)

MetricDefinisi
Availability rata-rata% uptime fleet dalam periode
Sedang DownCount event yang end_at masih NULL
MTTRMean Time To Repair — rata² lama perbaikan
MTBF (kalender)Rata² jarak antar kerusakan

Kategori Availability: 🟢 ≥95% 🟡 ≥85% 🔴 <85%

6. Workflow Catat Downtime — Hari 1 🔴

Skenario: equipment rusak, mekanik lapor segera

Aksi di app

  1. RAM → Availability → klik + Catat Downtime
  2. Isi: Equipment, Mulai Down (sekarang), Selesai (kosongkan — masih down)
  3. Penyebab: Breakdown / Troubleshoot
  4. Catatan: deskripsi singkat ("Engine knocking, dugaan turbo bocor")
  5. Section 📸 FOTO PENDUKUNG:
      • Fase: Sebelum / Bukti Kerusakan
      • Tgl kejadian foto: hari ini (default)
      • Caption: "Oli netes seal turbo"
      • Upload 1-5 foto kondisi rusak
  6. Simpan → toast "Upload N foto sukses"

7. Workflow Catat Downtime — Hari 3 📦

Skenario: part baru datang dari warehouse

Aksi di app

  1. RAM → Availability → list "🔴 Sedang Down" → klik Edit record
  2. Jangan ubah Mulai/Selesai (masih down)
  3. Section 📸 FOTO PENDUKUNG:
      • Fase: Part / Spare Part
      • Tgl kejadian foto: 29/05/2026 (manual, tanggal part datang)
      • Caption: "Turbo baru P/N HX35 dari gudang"
      • Upload foto part
  4. Simpan → foto ke-2 ditambah ke event yg sama, status tetap 🔴 Sedang Down
Kunci: Tgl kejadian foto bisa beda dari tgl upload → audit trail jujur kalau upload telat.

8. Workflow Catat Downtime — Hari 5 🟢

Skenario: perbaikan selesai, equipment kembali siap

Aksi di app

  1. Edit record yang sama lagi
  2. Isi Selesai = 31 Mei 2026, 17:00
  3. Section 📸 FOTO PENDUKUNG:
      • Fase: Sesudah / Bukti Perbaikan
      • Tgl kejadian: hari ini
      • Caption: "Pemasangan + test run OK"
      • Upload foto after
  4. Simpan → record closed, durasi auto-terhitung 4 hari 3 jam

Tampilan di History tab

3 baris timeline terpisah:
🟢 31 Mei • SELESAI PERBAIKAN (lama 4.1 hari) — Diselesaikan oleh: David — [foto after]
📦 29 Mei • Part datang / dipasang — Oleh: David — [foto part]
🔴 27 Mei • Penyebab: Breakdown — Dilaporkan oleh: M. Yusuf — [foto before]

9. Maintainability — PM, TOH, GOH

Sistem perawatan berbasis jam (HM) untuk engine/genset/pompa, atau km untuk kendaraan.

4 Tier OEM (industry-typical)

TierEngine classPMTOHGOHEco life
ASmall <80kW (Perkins 1103A, Cummins 4BTA)5006.00012.00030.000
BMedium 80-150kW (CAT C4.4, Deutz 1013)5008.00016.00035.000
CLarge 150kW+ (FG Wilson P220, CAT DE165ED)50010.00020.00040.000
DHeavy-Duty Rig (CAT D3406/C13/C15)50010.00020.00045.000

Semua dalam jam jalan. Sisa PM = (last_pm_hours + interval) − running_hours

10. Kelayakan Armada — Maintain vs Ganti

Gabungan 2 sumbu: umur (TOH/GOH/eco_life) + keandalan (MTBF).

VerdictAturan
🟢 SehatUmur <80% AND keandalan ≥1 siklus TOH
🟡 PantauSalah satu sumbu bermasalah
🔴 Kandidat gantiUmur ≥100% AND keandalan <½ TOH

Contoh nyata (kondisi sekarang)

GS-MTU-PERKINS: running_hours 36.679 jam, eco_life 30.000 jam
→ UMUR 122% → 🔴 Kandidat ganti ✋ direkomendasikan evaluasi penggantian

11. SOP Harian Mekanik

Morning check

  1. Cek dashboard eRAMHoist
  2. Lihat kartu PM Berbasis Jam/KM — ada unit hampir lewat?
  3. Lihat Sedang Down — perlu follow up?
  4. Update HM/KM unit yg baru jalan (Logbook "Isi HM" / Edit form)

Saat equipment rusak

  1. Catat Downtime baru segera (real-time) + foto kondisi rusak
  2. Multi-day repair: edit & update foto tiap milestone (part datang, dst)
  3. Setelah selesai: isi tanggal Selesai + foto after

Weekly review (Sr Mekanik)

Kelayakan: cek unit 🔴/🟡 watchlist · Availability: trend MTTR/MTBF · Reliability: pattern kerusakan

12. Common Pitfalls (jangan terjebak)

⚠️ Edit equipment butuh login admin
RLS equipment: write admin only. Edit via Logbook gagal diam-diam tanpa error kalau bukan admin. Selalu login admin saat seed/edit data master.
📊 HM Sekarang vs HM PM Terakhir
HM Sekarang = running_hours = odometer kumulatif (bukan jam harian)
HM PM Terakhir = last_pm_hours = baseline waktu PM terakhir
Sisa PM = (last_pm + interval) − running_hours
📸 Catatan vs Caption foto
Catatan downtime = deskripsi awal kerusakan (1× per event, di Catat Downtime form)
Caption foto = mini-log per upload (multiple foto, beda phase/tanggal)
